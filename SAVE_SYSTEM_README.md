# 保存系统使用说明

## 功能概述

本项目已添加完整的游戏保存/加载系统，支持以下功能：

### 1. 启动菜单
- 程序启动时会显示主菜单界面
- 如果存在存档，会显示"继续游戏"选项并显示存档信息（游戏时间、零件数量）
- 可以选择"新游戏"（会删除旧存档）或"退出"

### 2. 自动保存
- 游戏运行时每10秒自动保存一次（300帧）
- 保存时左下角会显示"AUTOSAVED"提示（持续3秒）
- 仅在发射后才开始自动保存（PRE_LAUNCH状态不保存）

### 3. 手动保存
- 游戏结束时（着陆或坠毁）会自动保存最终状态
- 控制台会显示"GAME SAVED"提示

### 4. 保存内容
保存文件包含以下数据：
- 火箭装配信息（所有零件及其配置）
- 火箭物理状态（位置、速度、姿态、燃料等）
- 控制输入状态（油门、姿态控制）
- 游戏时间和任务状态
- 当前影响球（SOI）信息

## 操作说明

### 启动菜单操作
- **上/下方向键**: 选择菜单选项
- **回车/空格**: 确认选择
- **ESC**: 退出游戏

### 菜单选项
1. **继续游戏** (CONTINUE GAME)
   - 仅在有存档时显示
   - 加载上次保存的游戏状态
   - 跳过火箭构建界面，直接进入飞行

2. **新游戏** (NEW GAME)
   - 开始新的游戏
   - 如果有旧存档，会提示"WARNING: WILL DELETE SAVE"
   - 进入火箭构建界面

3. **退出** (EXIT)
   - 关闭游戏

## 保存文件

### 文件位置
- 保存文件名: `rocket_save.dat`
- 位置: 游戏可执行文件所在目录

### 文件格式
保存文件使用文本格式，便于调试和手动编辑（不推荐）。
格式示例：
```
VERSION 1.0
PARTS 3
9 0
6 0
0 0
STATE
100000.0
0.0 6371100.0 0.0
...
```

## 技术细节

### 新增文件
1. `src/save_system.h` - 保存/加载核心功能
2. `src/menu_system.h` - 启动菜单界面和逻辑

### 修改文件
- `src/project.cpp` - 集成保存系统和菜单系统

### 主要函数
- `SaveSystem::SaveGame()` - 保存游戏状态
- `SaveSystem::LoadGame()` - 加载游戏状态
- `SaveSystem::HasSaveFile()` - 检查是否存在存档
- `SaveSystem::GetSaveInfo()` - 获取存档信息
- `MenuSystem::DrawMainMenu()` - 绘制主菜单
- `MenuSystem::HandleMenuInput()` - 处理菜单输入

## 注意事项

1. **存档兼容性**: 如果修改了游戏数据结构，旧存档可能无法加载
2. **保存频率**: 自动保存间隔为10秒，可在代码中修改（搜索"frame % 300"）
3. **存档安全**: 建议定期备份 `rocket_save.dat` 文件
4. **调试模式**: 保存文件使用文本格式，可以用文本编辑器查看内容

## 故障排除

### 问题: 无法加载存档
- 检查 `rocket_save.dat` 文件是否存在
- 检查文件是否损坏（可用文本编辑器打开查看）
- 尝试删除存档文件重新开始

### 问题: 自动保存不工作
- 确认已经发射火箭（PRE_LAUNCH状态不保存）
- 检查游戏目录是否有写入权限
- 查看控制台是否有错误信息

### 问题: 菜单显示异常
- 确认已正确编译所有新增文件
- 检查 OpenGL 初始化是否成功
- 尝试重新编译项目
